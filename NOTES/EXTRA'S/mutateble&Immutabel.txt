Mutability (Change hona) aur Immutability (Unmutate / Change na hona) poore JavaScript ka sabse bada rule hai, sirf Array ka nahi!

JavaScript mein saare data types 2 categories mein baante gaye hain:

A. Primitive Types (Always IMMUTABLE / Unmutate)
(String, Number, Boolean, null, undefined)
Ye values memory mein lock hoti hain. Aap inhe mutate nahi kar sakte.


let name = "Shiv";
name[0] = "R"; // Aapne socha 'Rhiv' ho jayega?
console.log(name); // Output: "Shiv" (Nahi badla! Kyunki String immutable hai)

// Sahi tarika: Aapko variable ko nayi value assign karni padegi
name = "Ram"; 


B. Reference Types (Always MUTABLE)
(Arrays, Objects, Functions)
Ye memory mein open rehte hain. Aap inke andar ka data directly badal (mutate) kar sakte hain.

EX: 1
const user = { name: "Shiv", age: 20 };
user.age = 21; // Original object mutate ho gaya!
console.log(user.age); // Output: 21

EX: 2
const numbers = [1, 2, 3];
numbers.push(4); 
console.log(numbers); // Output: [1, 2, 3, 4] (Original array badal gaya)

EX: 3
const original = [1, 2, 3];
// Humne spread operator se naya array banaya aur usme 4 daala
const newArray = [...original, 4]; 
 
EX: 4
console.log(original); // Output: [1, 2, 3] (Original safe hai!)
console.log(newArray); // Output: [1, 2, 3, 4]


MUTABLE AND UNMUTATE BY  Category DEVIDED--------------------------------------------------------------------------------------------------------------------------------
FOR ARRAY CHECK IN DETAILES :-----------------------------------------


🚨 Category 1: The Mutating Methods (Original Array Badal Jata Hai)
Ye wo methods hain jo aapke array ko andar se modify kar dete hain. 
Inhe use karte waqt dhyaan rakhna chahiye, 
especially agar aap React ya Redux use kar rahe hain.

Add / Remove Elements:

1.push(): Array ke end mein item add karta hai.
2.pop(): Array ke end se item hatata hai.
3.unshift(): Array ke shuru mein item add karta hai.
4.shift(): Array ke shuru se item hatata hai.
5.splice(): Array ke beech mein se items hatata ya add karta hai. (Yaad rakhein: Splice = Mutate).

Reorder / Modify Elements:
6.  sort(): Array ke items ko alphabetically ya number ke hisaab se sort karta hai.
7.  reverse(): Array ke order ko poora ulta kar deta hai.
8.  fill(): Array ke saare (ya kuch) elements ko kisi ek static value se bhar deta hai (e.g., arr.fill(0)).
9.  copyWithin(): Array ke hi kisi hisse ko copy karke usi array mein kisi aur jagah paste kar deta hai.

✅ Category 2: The Unmutating / Immutable Methods (Naya Array / Nayi Value Milti Hai)
Ye "Safe" methods hain. Ye original array ko kabhi hath nahi lagate.

Group A: Naya Array Return Karne Wale (Transformations)

1.map(): Har item ko modify karke ek naya array deta hai.
2.filter(): Condition ke hisaab se items ko chhan (filter) kar naya array deta hai.
3.slice(): Array ka ek chhota hissa kaat kar naya array deta hai. (Yaad rakhein: Slice = Safe).
4.concat(): Do ya zyada arrays ko jodkar ek naya array deta hai.
5.flat(): Agar array ke andar array hai [1, [2, 3]], toh usko flat karke [1, 2, 3] bana deta hai.
6.flatMap(): Pehle map() karta hai, phir usko flat() kar deta hai.

Group B: The New ES2023 Methods (Modern & Safe Replacements)
Purane mutating methods ke ab safe versions aa gaye hain:
7.  toSorted(): sort() ka safe version.
8.  toReversed(): reverse() ka safe version.
9.  toSpliced(): splice() ka safe version.
10. with(index, value): Kisi specific index ki value badal kar ek naya array return karta hai (e.g., arr.with(1, "Apple")).

Group C: Single Value ya Boolean Return Karne Wale (Evaluation / Searching)
IMMUTABLE;;
Ye methods array mein se kuch dhoondhte hain ya calculate karte hain. Ye bhi original ko nahi badalte.

11. reduce() / reduceRight(): Poore array ko calculate karke ek single value deta hai (jaise total sum).
12. includes(): Check karta hai ki koi item array mein hai ya nahi (true ya false).
13. indexOf() / lastIndexOf(): Kisi item ki position (index number) batata hai.
14. find(): Condition match karne wala pehla item deta hai.
15. findIndex(): Condition match karne wale pehle item ka index deta hai.
16. findLast() / findLastIndex(): Array ke end se dhoondhna shuru karte hain.
17. some(): Check karta hai ki kya array mein kam se kam ek item condition pass karta hai (true/false).
18. every(): Check karta hai ki kya array ke saare items condition pass karte hain (true/false).
19. join(): Array ke saare items ko jodkar ek String bana deta hai.

Group D: Iterators (Looping)
20. forEach(): Array ke har item par ek function chalata hai. 
Ye directly array ko mutate nahi karta (lekin aap chahain toh iske andar code likh kar mutate karwa sakte hain). 
Ye hamesha undefined return karta hai.
